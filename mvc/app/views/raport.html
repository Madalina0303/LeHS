Introduction

Proiectul LeHS are ca scop exersarea si verificarea cunostintelor despre Html si Css astfel incat utilizatorul sa dobandeasca noi abilitati pentru scrierea de cod intr-o maniera interactiva,prin joc.

Application Arhitecture

Front End

Paginile site-ului web sunt responsive iar tematica jocului este reprezentata de o jungla cu animale Interfata este una atractiva si captivanta, functioanalitatile aplicatiei fiind usor de accesat si utilizat.

Sign Up & Login

Prima pagina a aplicatiei este "index.html" prin care utilizatorul poate accesa jocul fie prin logare fie prin inregistrare. Sunt disponibile optiunile de conectare folosind conturile de github,. Navigarea intre sectiunile de logare respectiv inregistrare se realizeaza prin glisarea unui buton iar butonul de sumbit al formularului de inregistrare necesita completarea tuturor campurilor cu date valide. Printre functioanalitati se numara si acceptarea termenilor si conditiilor pentru prelucrarea datelor personale. Deasemenea de pe aceasta pagina utilizatorul are acces la galeria jocului.

Game Gallery

Galeria jocului, identificata prin pagina "gallery.php", poate fi accesata de pe pagina principala si pune la dispozitie o serie de imagini ilustrative pentru tematica jocului, creand o perspectiva de ansamblu asupra caracterelor ce vor insoti utilizatorul in experienta traita pe insula. Utilizatorul poate deschide o imagine ce va fi dispusa pe intregul ecran si poate naviga intre imagini folosindu-se de cele doua butoane prev si next.

Menu

Se pot selecta diverse pagini ale aplicatiei cum ar fi : selectarea nivelului de joc, vizualizarea unui clasament, a profilului,setari respectiv inceperea/continuarea exercitiilor. Pagina de meniu este realizata intr-o maniera unica,personalizata prin modul serial de dispunere al sectiunilor.

Levels

Din meniu se ajunge pe pagina "levels.html" unde utilizatorul poate selecta o categorie pentru care opteaza sa-si verifice cunostintele precum si un nivel de dificultate. Butoanele corespunzatoare categoriilor( Html sau Css ) au o forma circulara si contin descrieri ale nivelelor. La trecerea mouse-ului peste buton se schimba background-ul iar optiunea de a incepe nivelul substituieste descrierea initiala. La partea de html pentru fiecare provocare se propune jucatorului o serie de 4 exercitii. Un exercitiu efectuat corect il deblocheza pe urmatorul. In cazul celui de-al 4 lea exercitiu devine disponibila provocarea urmatoare. Un personaj/caracter este prezent si afiseaza un mesaj prin glisarea mouse-ului pe deasupra. Legaturile catre paginile de selectare a nivelului, de vizitare a profilului si a meniului fac posibila navigarea catre cele mai importante sectiuni ale aplicatiei. La partea de Css este doar cate un exercitiu per provocare.

Profile

In aceasta pagina, utilizatorul isi poate vedea progresul corespunzator fiecarei categorii de exercitii (Html, Css, All), nivelul curent si cat mai are de parcurs pana la urmatorul nivel. Este afisata si fotografia utilizatorului, username-ul si optiunea de navigare catre pagina de cadouri/avantaje in joc pentru utilizator.

Leader Board

In aceasta pagina, utilizatorul are acces la clasamentul utilizatorilor precum si locul pe care este situat raportat la ceilalti jucatori. Topul jucatorilor este structurat in 3 sectiuni. Acestea din urma find in stransa legatura cu categoriile jocului( Html, Css, All).

Settings

Setari cu privire la sunetul de fundal respectiv a suportului pentru internationalizare( limbile: engleza si franceza) pentru a putea fi accesat pe o scara larga de utilizatori. Fondul muzical are ca scop cresterea intesitatii si suspansului, mentinand interesul si atentia utilizatorului.

Data Base

In baza de date exista tabele precum : userTable in care se pastreaza informatiile corespunzatoare logarii, cum ar fi username ul, adresa de email, avatarul Alte tabele sunt cele in care retinem nivelul si provocarea fiecarui utilizator astfel incat la revenirea in joc sa continue de unde a ramas
